# v0.2 Release Readiness

This is an internal release-readiness checklist for `thalmetis-edr` v0.2. It
tracks release preparation without itself executing publication, tagging,
archiving, PyPI publishing, or post-release DOI metadata updates.

## Scope

v0.2 release readiness covers the bounded McRae 2024 Figure 5a / Table 3
scientific core after the v0.2 primitives landed:

- packaged Figure 5a affected-volume interpolation
- packaged inferred Table 3 calculator `R_b` interpolation
- bounded pinch-off viability sensitivity estimates using Figure 5a affected
  volume, optional inferred Table 3 `R_b`, gas-exposure normalization,
  spherical bubble-volume/event-count arithmetic, and McRae Equation 3

v0.2 remains research and educational methodology tooling. It is not a
validated industrial viability predictor, process-control tool, GMP release
system, transfer-approval engine, batch-release tool, or Track A commercial
transfer-readiness product.

## Explicit Exclusions

- no new scientific model behavior
- no Equation 2 operational model
- no extrapolation
- no rupture viability
- no coalescence
- no path-history or path-independence behavior
- no thesis Table 4.4
- no sensitivity grid
- no notebook
- no Track A transfer-readiness integration
- no generated demo artifact changes
- no hosted/web/server/cloud API behavior
- no process-control, GMP, transfer-approval, process-approval, batch-release,
  or validated industrial prediction claims
- no PyPI publishing automation
- no full-repo formatting cleanup

## README / Docs / Examples / CI Checklist

- README describes v0.1 reconstruction and v0.2 bounded interpolation plus
  sensitivity estimates.
- README uses source installation instructions because PyPI publication is
  deferred.
- README examples prefer user-supplied bubble radius when available.
- README and docs identify inferred Table 3 `R_b` as calculator values from
  the v0.1 reconciliation, not measured bubble-size data.
- Model-boundary docs state users supply EDR threshold and single-event
  viability-loss assumptions.
- References include McRae et al. 2024 and Walls et al. 2017 DOIs.
- Example scripts smoke-run without plots or new dependencies.
- CI runs both Table 3 and pinch-off viability example scripts.

## Tests / Lint Checklist

Before release approval, confirm:

- local `main` is clean and synced with `origin/main`
- `.venv\Scripts\python.exe -m pytest` passes
- `.venv\Scripts\python.exe -m ruff check .` passes
- `.venv\Scripts\python.exe examples/table3_sensitivity.py` passes
- `.venv\Scripts\python.exe examples/pinchoff_viability_sensitivity.py` passes
- GitHub Actions CI is green for the release candidate

Full-repo `ruff format --check .` may still report known untouched formatting
drift. That drift should be handled in a separate mechanical housekeeping PR,
not mixed into v0.2 release materials.
